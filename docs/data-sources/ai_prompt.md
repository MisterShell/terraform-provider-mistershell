# mistershell_ai_prompt (Data Source)

Reads a MisterShell AI prompt. Look up by id for a direct fetch, or by name (exact match). Both custom (type="user") and builtin (type="system") prompts are readable here.

## Example Usage

```terraform
# Look up an AI prompt by its integer ID.
data "mistershell_ai_prompt" "by_id" {
  id = 42
}

# Look up an AI prompt by its exact name.
# This works for both user-defined prompts and builtin "system" prompts:
# the builtin system prompts are read-only and only readable through the
# data source (they cannot be created or managed as a resource).
data "mistershell_ai_prompt" "general_assistant" {
  name = "builtin.general_assistant"
}

output "prompt_type" {
  value = data.mistershell_ai_prompt.general_assistant.type # => "system"
}

output "prompt_content" {
  value = data.mistershell_ai_prompt.general_assistant.content
}
```

## Schema

### Optional

- `id` (Number) AI prompt ID. Use for direct lookup.
- `name` (String) AI prompt name. Used as an exact-match search filter when id is not specified.

### Read-Only

- `content` (String) The prompt text. May contain Jinja2 template variables such as {{resource_id}}.
- `created_at` (String) Creation timestamp.
- `description` (String) Human-readable description of the prompt.
- `type` (String) Prompt type. Either "user" (custom) or "system" (builtin/read-only).
- `updated_at` (String) Last update timestamp.
- `variable_schema` (String) JSON Schema documenting the template variables used in content.

## Lookup arguments

Specify exactly one of:

| argument | behavior |
|---|---|
| `id` | Direct lookup by integer ID. |
| `name` | Exact-match lookup by name (fuzzy server search, then exact `name ==` filter). Errors if zero or more than one prompt matches. |

The data source reads **both** `user` prompts and builtin `system` prompts. The
builtin system prompts (read-only, created from the backend prompt registry) can
only be inspected this way — they are not manageable through the
[`mistershell_ai_prompt` resource](../resources/ai_prompt.md). Their names are
`builtin.general_assistant`, `builtin.resource_summary`, `builtin.config_analysis`,
`builtin.session_assist`, `builtin.session_analysis`, and `builtin.executive_summary`.

For the full set of `type` values, the builtin prompt catalog, and the Jinja2

# mistershell_ai_model (Data Source)

Reads a MisterShell AI model. Look up by id for a direct fetch, or by name (exact match). Note: provider secrets in config (e.g. api_key) are masked as '***' by the API.

## Example Usage

```terraform
# Look up an AI model by its integer ID.
data "mistershell_ai_model" "by_id" {
  id = 123
}

# Look up an AI model by its exact name.
data "mistershell_ai_model" "by_name" {
  name = "claude-sonnet"
}

output "model_provider" {
  value = data.mistershell_ai_model.by_name.model_provider
}

output "model_id" {
  value = data.mistershell_ai_model.by_name.model_id
}

output "model_is_default" {
  value = data.mistershell_ai_model.by_name.is_default
}
```

## Schema

### Optional

- `id` (Number) AI model ID. Use for direct lookup.
- `name` (String) AI model name. Used as an exact-match search filter when id is not specified.

### Read-Only

- `config` (String, Sensitive) Per-provider config as JSON. Provider secrets (e.g. api_key) are masked as '***' by the API.
- `created_at` (String) Creation timestamp.
- `is_default` (Boolean) Whether this is the default AI model.
- `model_id` (String) Provider-specific model identifier.
- `model_provider` (String) AI provider backing this model (anthropic, azure_openai, bedrock, cohere, google, mistral, ollama, openai, openrouter, xai).
- `updated_at` (String) Last update timestamp.

## Lookup arguments

Specify exactly one of:

| argument | behavior |
|---|---|
| `id` | Direct lookup by integer ID. |
| `name` | Exact-match lookup by name (fuzzy server search, then exact `name ==` filter). Errors if zero or more than one model matches. |

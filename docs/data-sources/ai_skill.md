# mistershell_ai_skill (Data Source)

Reads a MisterShell AI skill (markdown platform brief surfaced to agents via list_skills). Look up by id for a direct fetch, or by name (exact match). Both user-defined and builtin skills (is_builtin=true) are exposed read-only here.

## Example Usage

```terraform
# Look up an AI skill by its integer ID.
data "mistershell_ai_skill" "by_id" {
  id = 42
}

# Look up an AI skill by its exact name.
# This works for both custom skills and builtin (MisterShell-managed) skills:
# builtin skills (is_builtin = true) are read-only and only readable through the
# data source â€” they cannot be created or managed as a resource.
data "mistershell_ai_skill" "linux_brief" {
  name = "linux-platform-brief"
}

output "skill_is_builtin" {
  value = data.mistershell_ai_skill.linux_brief.is_builtin # => true
}

output "skill_resource_types" {
  value = data.mistershell_ai_skill.linux_brief.resource_types # => ["linux"]
}
```

## Schema

### Optional

- `id` (Number) AI skill ID. Use for direct lookup.
- `name` (String) Skill name. Used as an exact-match search filter when id is not specified.

### Read-Only

- `agent_types` (Set of String) Agent types the skill discovery is restricted to (null if unrestricted).
- `body` (String) Markdown platform-brief content surfaced to agents.
- `created_at` (String) Creation timestamp.
- `description` (String) Human-readable description of the skill.
- `is_builtin` (Boolean) Whether the skill is a builtin (managed by MisterShell). Builtin skills are read-only except their is_enabled flag.
- `is_enabled` (Boolean) Whether the skill is enabled.
- `resource_types` (Set of String) Resource-type keys the skill discovery is restricted to (null if unrestricted).
- `updated_at` (String) Last update timestamp.

## Lookup arguments

Specify exactly one of:

| argument | behavior |
|---|---|
| `id` | Direct lookup by integer ID. |
| `name` | Exact-match lookup by name (fuzzy server search, then exact `name ==` filter). Errors if zero or more than one skill matches. |

This data source reads **both** custom skills and builtin
(MisterShell-managed) skills. Builtin skills (`is_builtin = true`) are read-only
and can only be read here â€” they cannot be created or managed as a resource.

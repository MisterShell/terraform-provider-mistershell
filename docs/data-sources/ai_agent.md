# mistershell_ai_agent (Data Source)

Reads a MisterShell AI agent (user-defined or builtin). Look up by id for a direct fetch, or by name (exact match).

## Example Usage

```terraform
# Look up an AI agent by its integer ID.
data "mistershell_ai_agent" "by_id" {
  id = 123
}

# Look up an AI agent by its exact name. This also works for builtin agents,
# which are managed by MisterShell and exposed read-only here
# (e.g. "General Assistant", "Resource Summary", "Session Assist").
data "mistershell_ai_agent" "by_name" {
  name = "General Assistant"
}

output "agent_type" {
  value = data.mistershell_ai_agent.by_name.type
}

output "agent_is_builtin" {
  value = data.mistershell_ai_agent.by_name.is_builtin
}

output "agent_is_functional" {
  value = data.mistershell_ai_agent.by_name.is_functional
}

output "agent_tool_ids" {
  value = data.mistershell_ai_agent.by_name.tool_ids
}
```

## Schema

### Optional

- `id` (Number) AI agent ID. Use for direct lookup.
- `name` (String) Agent name. Used as an exact-match search filter when id is not specified.

### Read-Only

- `config` (String) Agent config as JSON.
- `created_at` (String) Creation timestamp.
- `description` (String) Agent description.
- `is_builtin` (Boolean) Whether the agent is a builtin agent managed by MisterShell.
- `is_functional` (Boolean) Whether the agent is currently functional (model and prompt resolve).
- `model_id` (Number) FK to the AI model used by the agent (null means the default model).
- `system_prompt_id` (Number) FK to the AI prompt providing the agent's system prompt.
- `tool_ids` (Set of Number) Tool IDs the agent may use; empty means ALL tools are allowed.
- `type` (String) Agent type (chat, background, or a builtin_* type).
- `updated_at` (String) Last update timestamp.

## Lookup arguments

Specify exactly one of:

| argument | behavior |
|---|---|
| `id` | Direct lookup by integer ID. |
| `name` | Exact-match lookup by name (fuzzy server search, then exact `name ==` filter). Errors if zero or more than one agent matches. |

This data source resolves **both** user agents (`chat`, `background`) and
builtin agents (`builtin_*`), which are otherwise read-only and not manageable
through the resource.

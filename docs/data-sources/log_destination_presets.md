# mistershell_log_destination_presets (Data Source)

Reads the MisterShell log-destination presets (read-only). Presets are code-owned per-type/vendor config templates (key, label, vendor, type, default_config). Copy a preset's default_config into a mistershell_log_destination's config to author a destination quickly.

## Example Usage

```terraform
# Read the code-owned log-destination presets.
data "mistershell_log_destination_presets" "all" {}

# All available preset keys.
output "preset_keys" {
  value = [for p in data.mistershell_log_destination_presets.all.presets : p.key]
}

# Build a destination from the Splunk HEC preset's default_config, overriding
# the URL and the (secret) header value.
locals {
  splunk = one([
    for p in data.mistershell_log_destination_presets.all.presets : p
    if p.key == "splunk_hec"
  ])
}

resource "mistershell_log_destination" "splunk" {
  name    = "splunk-hec"
  type    = local.splunk.type
  streams = ["security"]

  config = jsonencode(merge(jsondecode(local.splunk.default_config), {
    url = "https://splunk.example.com:8088/services/collector"
    auth = {
      type         = "header"
      header_name  = "Authorization"
      header_value = "Splunk ${var.splunk_token}"
    }
  }))
}

variable "splunk_token" {
  type      = string
  sensitive = true
}
```

## Overview

Log-destination presets are a **fixed, code-owned registry** maintained in the
product, not by Terraform. This data source is read-only and takes no arguments;
it returns every preset the instance defines. Each preset is a ready-to-edit
`config` template for a particular vendor/type — copy a preset's `default_config`
into a [`mistershell_log_destination`](../resources/log_destination.md) `config`
to author a destination quickly (see the Example Usage above).

## Available presets

The registry currently defines five presets:

| key | label | vendor | type | default_config highlights |
|---|---|---|---|---|
| `custom_syslog` | Custom Syslog | Custom | `syslog` | `host` empty, `port` 514, `protocol` `UDP`, `format` `RFC5424`, `facility` `local0`, `tls_verify` true |
| `custom_webhook` | Custom Webhook | Custom | `webhook` | `url` empty, `method` `POST`, `headers` `{}`, `auth` `{type:none}`, `body_format` `raw`, `timeout_seconds` 5, `tls_verify` true |
| `splunk_hec` | Splunk HEC | Splunk | `webhook` | `url` `…:8088/services/collector`, `auth` `{type:header, header_name:Authorization, header_value:""}`, `body_format` `splunk_hec` |
| `elastic` | Elastic | Elastic | `webhook` | `url` empty, `auth` `{type:header, header_name:Authorization, header_value:""}`, `body_format` `raw` |
| `datadog` | Datadog | Datadog | `webhook` | `url` `…/api/v2/logs`, `auth` `{type:header, header_name:DD-API-KEY, header_value:""}`, `body_format` `raw` |

The live set (including each `default_config` verbatim) is returned in the
`presets` attribute. For the full per-type `config` field reference, see the
[`mistershell_log_destination` resource](../resources/log_destination.md) page.

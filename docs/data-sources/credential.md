# mistershell_credential (Data Source)

Reads a MisterShell credential. Look up by id for a direct fetch, or use name/credential_type to search. Search filters must match exactly one credential. Note: secret fields in credential_data are masked as '****' by the API.

## Example Usage

```terraform
# Look up by ID
data "mistershell_credential" "by_id" {
  id = 1
}

# Look up by unique name
data "mistershell_credential" "by_name" {
  name = "dc-admin-ssh"
}

# Look up by type (must match exactly one)
data "mistershell_credential" "k8s" {
  credential_type = "kubeconfig"
}
```

## Schema

### Optional

- `credential_type` (String) Credential type filter (ssh_password, ssh_key, aws_credentials, azure_service_principal, kubeconfig, rdp_password, db_password).
- `id` (Number) Credential ID. Use for direct lookup.
- `name` (String) Credential name (unique). Used as a search filter when id is not specified.

### Read-Only

- `created_at` (String) Creation timestamp.
- `credential_data` (String, Sensitive) Credential data as JSON. Secret fields are masked as '****' by the API.
- `description` (String) Credential description.
- `extra_data` (String) Non-sensitive metadata as JSON.
- `requires_user_mapping` (Boolean) Whether users must fill their own credential copy.
- `updated_at` (String) Last update timestamp.

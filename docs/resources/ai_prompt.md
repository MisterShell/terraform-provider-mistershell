# mistershell_ai_prompt (Resource)

Manages a custom MisterShell AI prompt (type="user"). Builtin type="system" prompts cannot be created, edited, or deleted (the API returns 403) and are available through the mistershell_ai_prompt data source.

## Example Usage

```terraform
# A user-defined prompt whose content references a Jinja2 template variable.
# Variables are written as {{ variable }} and are substituted by the backend
# when an agent runs the prompt (see the resource docs for the full variable set).
resource "mistershell_ai_prompt" "resource_summary" {
  name = "concise-resource-summary"
  type = "user" # only "user" is allowed; "system" prompts are builtin/read-only

  content = <<-EOT
    Summarize resource {{ resource_id }} ("{{ resource_name }}", type {{ resource_type }}) concisely.
    Today is {{ current_date }}. Highlight only anomalies and the single most important follow-up.
  EOT

  description = "Short, anomaly-focused resource summary used by a custom agent."

  # Optional JSON Schema documenting the variables this prompt expects.
  # Stored from configuration; informational only (the backend does not
  # reject undeclared variables based on this schema).
  variable_schema = jsonencode({
    type = "object"
    properties = {
      resource_id   = { type = "integer", description = "Target resource ID." }
      resource_name = { type = "string", description = "Resource display name." }
      resource_type = { type = "string", description = "Resource type descriptor key." }
    }
    required = ["resource_id"]
  })
}
```

## Builtin (system) prompts

The backend seeds a fixed set of `type = "system"` prompts from its prompt
registry on startup and keeps them in sync. These are read-only: any attempt to
create, modify, or delete them returns **HTTP 403**. You can read them by name
through the data source. The builtin system prompt names are:

| name | purpose |
|---|---|
| `builtin.general_assistant` | Default system prompt for the General Assistant chat agent. |
| `builtin.resource_summary` | Health summary prompt for a MisterShell resource. |
| `builtin.config_analysis` | Configuration analysis (diff between two snapshot versions, or initial-config description). |
| `builtin.session_assist` | Live remote-session assistant prompt. |
| `builtin.session_analysis` | Analysis of a completed remote session recording. |
| `builtin.executive_summary` | Executive summary at the top of an operational report. |

## Template variables

`content` is rendered as a Jinja2 template at agent-run time, so it may contain
`{{ variable }}` placeholders (and Jinja2 control blocks such as
`{% if snapshot_id %} ... {% endif %}`). Substitution happens
when an agent executes the prompt; the value of each variable comes from that
agent's run context. **The set of available variables therefore depends on which
agent the prompt is attached to** — a variable that is undefined in a given
context simply renders empty (guard optional variables with `{% if ... %}`).

Two **universal** variables are injected for every prompt regardless of agent:

| variable | type | description |
|---|---|---|
| `current_date` | datetime | Current ISO date at agent-preparation time (UTC). |
| `current_time` | datetime | Current ISO time at agent-preparation time (UTC). |

## The `variable_schema` field

`variable_schema` is an **optional** JSON Schema (a JSON object set with
`jsonencode(...)`) that documents the variables a prompt expects — names, types,
and which are required. It is advisory metadata for the prompt editor; it does
**not** gate rendering (an undeclared variable still renders, and a missing
optional one renders empty).

As with other stored-from-config JSON fields, the provider stores
`variable_schema` **from your configuration** and does not read it back from the
server. Consequently it is **excluded from import** (only `id` is set on import);
re-supply it in HCL after importing so the next plan shows no diff.

**Deletion is blocked while referenced.** Deleting a prompt that is still
attached to one or more agents (as their `system_prompt`) returns **HTTP 403**
("Cannot delete prompt linked to agents: ..."). Detach the prompt from every
agent first.

## Import

Import is supported using the following syntax:

```shell
# AI prompts are imported by their integer ID.
# Note: variable_schema is stored from your configuration and is NOT set on
# import (only id is). Re-supply variable_schema in your configuration after
# importing so the next plan does not show a diff.
terraform import mistershell_ai_prompt.example 42
```

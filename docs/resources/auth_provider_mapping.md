# mistershell_auth_provider_mapping (Resource)

Manages a MisterShell external group->role mapping under an auth provider. (provider_id, external_group) is unique. Import format is "<provider_id>:<mapping_id>". Create and update require an active license; deleting the parent provider cascades to its mappings.

## Example Usage

```terraform
# Maps an external group (OIDC claim value here) to a MisterShell role.
# (provider_id, external_group) is unique. Create and update require an active
# license.
resource "mistershell_auth_provider_mapping" "admins" {
  provider_id    = mistershell_auth_provider.corp_oidc.id
  external_group = "platform-admins"
  role_id        = mistershell_role.admin.id
}
```

## Schema

### Required

- `external_group` (String) External group identifier (LDAP DN / OIDC claim value / SAML attribute value), 1-500 characters.
- `provider_id` (Number) ID of the auth provider this mapping belongs to. Cannot be changed after creation.
- `role_id` (Number) ID of the MisterShell role granted to members of the external group.

### Read-Only

- `created_at` (String) Creation timestamp.
- `id` (Number) Mapping ID.
- `role_name` (String) Name of the mapped role (read-only).
- `updated_at` (String) Last update timestamp.

## Field reference

| attribute | constraints |
|---|---|
| `provider_id` | Required integer. ID of the parent `mistershell_auth_provider`. **Immutable** — changing it forces replacement. |
| `external_group` | Required. 1-500 characters. The external group identifier: an LDAP DN, an OIDC claim value, or a SAML attribute value, depending on the parent provider's type. |
| `role_id` | Required integer. ID of the `mistershell_role` granted to members of `external_group`. |
| `role_name` | Read-only. Name of the mapped role. |

## Notes

- **`(provider_id, external_group)` is unique.** Two mappings with the same
  provider and external group are rejected by the API (`409`).
- **License required.** Create and update require an active license; an unlicensed
  instance returns a `403` error, surfaced verbatim. Delete works unlicensed.
- **Cascade delete.** Deleting the parent `mistershell_auth_provider` cascades to
  its mappings; the provider tolerates a mapping that is already gone.

## Import

Mappings are nested under a provider, so the import ID is a **compound**
`"<provider_id>:<mapping_id>"` (not a bare integer).

Import is supported using the following syntax:

```shell
# Group mappings are nested under a provider, so the import ID is a compound
# "<provider_id>:<mapping_id>".
terraform import mistershell_auth_provider_mapping.admins 3:11
```

# mistershell_location (Resource)

Manages a MisterShell location. Locations form a hierarchy for organizing network resources by geography or organizational structure.

## Example Usage

```terraform
# Create a location hierarchy: EMEA > Switzerland > Zurich
resource "mistershell_location" "emea" {
  name        = "EMEA"
  kind        = "geo"
  description = "Europe, Middle East, and Africa"
}

resource "mistershell_location" "switzerland" {
  name      = "Switzerland"
  kind      = "geo"
  parent_id = mistershell_location.emea.id
}

resource "mistershell_location" "zurich" {
  name        = "Zurich DC"
  kind        = "geo"
  description = "Zurich data center"
  parent_id   = mistershell_location.switzerland.id
  latitude    = 47.3769
  longitude   = 8.5417
}
```

## Schema

### Required

- `name` (String) Location name.

### Optional

- `description` (String) Location description.
- `extra_data` (String) Additional metadata as JSON. Use jsonencode() in HCL.
- `kind` (String) Location type: 'geo' for geographic or 'org' for organizational.
- `latitude` (Number) Geographic latitude.
- `longitude` (Number) Geographic longitude.
- `parent_id` (Number) Parent location ID for hierarchical nesting.

### Read-Only

- `created_at` (String) Creation timestamp.
- `id` (Number) Location ID.
- `updated_at` (String) Last update timestamp.

## Valid `kind` values

| kind | meaning |
|---|---|
| `geo` | Geographic location (region/site/datacenter). Default when `kind` is omitted. |
| `org` | Organizational unit. |

Omitting `parent_id` makes the location a **root** location. Root locations
cannot be deleted by the API.

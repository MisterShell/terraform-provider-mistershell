# mistershell_log_destination (Resource)

Manages a MisterShell log destination, forwarding security/policy/api/app log streams to a syslog server or webhook endpoint. The config attribute is an opaque per-type JSON blob (syslog or webhook); webhook auth secrets are masked by the API and stored from config (not read back).

## Example Usage

```terraform
# Syslog destination over TLS, forwarding security and policy logs.
resource "mistershell_log_destination" "syslog" {
  name         = "central-syslog"
  type         = "syslog"
  enabled      = true
  streams      = ["security", "policy"]
  min_severity = "medium"

  config = jsonencode({
    type       = "syslog"
    host       = "logs.example.com"
    port       = 6514
    protocol   = "TLS"
    format     = "RFC5424"
    facility   = "local1"
    tls_verify = true
  })
}

# Webhook destination with bearer-token auth, forwarding all streams.
resource "mistershell_log_destination" "webhook" {
  name         = "siem-webhook"
  type         = "webhook"
  enabled      = true
  streams      = ["security", "policy", "api", "app"]
  min_severity = "info"

  config = jsonencode({
    type            = "webhook"
    url             = "https://siem.example.com/ingest"
    method          = "POST"
    body_format     = "raw"
    timeout_seconds = 5
    tls_verify      = true
    headers = {
      "X-Source" = "mistershell"
    }
    auth = {
      type  = "bearer"
      token = var.webhook_token # secret; masked by the API, stored from config
    }
  })
}

variable "webhook_token" {
  type      = string
  sensitive = true
}
```

## The `config` JSON blob

`config` is an opaque, per-`type` JSON object written with `jsonencode(...)`. Its
shape is determined by `type` and must include a matching `type` discriminator
field inside the blob. The fields below are the complete set the backend accepts;
any other key is rejected.

**Secret handling.** Webhook `auth` secret fields (marked below) are encrypted at
rest and **masked to `****`** in every API response. As with `credential_data`,
the provider stores `config` **from your configuration** and does **not** read it
back from the server. Two consequences:

- After **importing** a webhook destination, its secret is unknown until you
  re-supply the full `config` in HCL — a webhook secret cannot be recovered from
  the masked server value.
- For webhook destinations, out-of-band changes to the **non-secret** parts of
  `config` (url, method, headers, etc.) are **not** detected as drift, because the
  whole `config` is preserved from state. This is the documented trade-off of the
  stored-from-config rule (identical to `credential_data`).
- `config` is **excluded from import** (only `id` is set on import); re-supply it
  in HCL, exactly like `credential_data`.

#### Webhook `auth` variants

`auth` is itself a discriminated object keyed by `auth.type`. Choose one of the
four types below. The **secret** fields are masked to `****` by the API and stored
from config (see the secret-handling note above).

| auth.type | fields | secret field(s) |
|---|---|---|
| `none` | `type` only | _(none)_ |
| `bearer` | `type`, `token` (minLength 1) | **`token`** |
| `basic` | `type`, `username` (1-255), `password` (minLength 1) | **`password`** |
| `header` | `type`, `header_name` (1-255), `header_value` (minLength 1) | **`header_value`** |

Example (bearer auth):

```hcl
config = jsonencode({
  type            = "webhook"
  url             = "https://siem.example.com/ingest"
  method          = "POST" # POST | PUT
  body_format     = "raw"  # raw | splunk_hec
  timeout_seconds = 5      # 1..60
  tls_verify      = true
  headers = {
    "X-Source" = "mistershell"
  }
  auth = {
    type  = "bearer"
    token = var.webhook_token # secret; masked by the API, stored from config
  }
})
```

Example (basic auth):

```hcl
config = jsonencode({
  type = "webhook"
  url  = "https://siem.example.com/ingest"
  auth = {
    type     = "basic"
    username = "svc-logs"
    password = var.webhook_password # secret
  }
})
```

## Import

Import is supported using the following syntax:

```shell
# Log destinations are imported by their integer ID.
# Note: webhook auth secrets (bearer token, basic password, header value) are
# NOT read back from the server (the API masks them as "****"); set the full
# config in your configuration after importing.
terraform import mistershell_log_destination.example 123
```

# mistershell_tag (Resource)

Manages a MisterShell tag. Tags are free-form labels (name, color, description) that can be assigned to network resources. The optional resource_ids set manages, with whole-set replace semantics, exactly which network resources carry this tag.

## Example Usage

```terraform
# A tag with a custom color and a description.
resource "mistershell_tag" "production" {
  name        = "production"
  color       = "#e53935"
  description = "Production network resources"
}

# A tag that owns the full set of resources it is assigned to.
# Managing resource_ids takes exclusive ownership of the tag's membership:
# Terraform replaces the whole set on every change.
resource "mistershell_tag" "core_switches" {
  name        = "core-switches"
  color       = "blue"
  description = "Core switching layer"

  resource_ids = [
    mistershell_resource.spine1.id,
    mistershell_resource.spine2.id,
  ]
}
```

### `color`

`color` is a **free-form string**, 1-16 characters, defaulting to `grey`. There
is **no fixed palette and no enum** — any string within the length limit is
accepted (a named color such as `grey`, `blue`, `red`, or a hex value such as
`#e53935`). Because it has a server-side default, `color` is Optional and
Computed: omit it and it resolves to `grey`.

### `resource_ids`

`resource_ids` is the **complete set** of network-resource IDs that carry this
tag. It maps to the tag's assignment endpoint, which uses **whole-set replace
semantics**: on every apply Terraform sends the entire planned set, replacing
whatever was assigned before. Managing this attribute therefore takes
**exclusive ownership** of the tag's resource membership — IDs added out of band
(through the UI or another caller) are removed on the next apply, and IDs you
remove from the set are unassigned.

Every ID must reference an existing network resource; an unknown ID is rejected
by the server. The attribute is a Set (order-independent). Omit it entirely to
leave membership unmanaged — the provider reads the current set back into state
but never writes it.
